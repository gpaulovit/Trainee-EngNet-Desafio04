# Padrão de Commits e Branching

## Branching

- `main` — código estável pronto para entrega
- `developer` — integração de funcionalidades em andamento
- `feature/xxx` — novas funcionalidades
- `hotfix/xxx` — correções urgentes

## Commits

- Usar mensagens claras e estruturadas:
  - `feat:` nova funcionalidade
  - `fix:` correção de bug
  - `docs:` documentação
  - `chore:` tarefas de manutenção
